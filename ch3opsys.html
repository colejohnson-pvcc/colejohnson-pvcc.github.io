Operating Systems Terminology
TERM DEFINITION
operating system The software that controls the overall operation of the computer. This includes the file management, the user interface, and the program execution environment.
job queue The organization of computer tasks or jobs residing in the mass storage of the computer awaiting execution.
FIFO FIFO stands for "first-in, first-out" which is the order that jobs are completed in a job queue.
interactive processing A feature which allowes operating systems that are exectuing a program to carry on a diolgue with the user through remote terminals.
time-sharing multiprogramming An operating system that provides service to multiple users at the same time by dividing that time into intervals and executing the job in the restricted interval.
application software Application software consists of programs that perform tasks particular to the machine's utlization. This includes spreadsheets, database systems, desktop publishing systems, accounting systems, program delelopment software, and games.
system software System software performs the tasks that are common to computer systems and provides the infrastructure that the application software requires.
utility software Software units located in the system software that consist of programs for performing activities that are fundamental to computer installations but not in the operating system. This includes software to compress and decompress data, for playing multimedia content, for handling network communication, for system software to be customized, to format a disk, and to copy a file from a magnetic disk to a flash drive.
graphical user interface (GUI) Systems that allow users to issue commands to files or programs and are represented pictorally on the display as icons.
shell Older user interfaces that communictated with users through textual messages using a keyboard and monitor screen.
kernel The internal part of an operating system. It contains the sofware components that perform the very basic functions required by the computer installation.
device driver A collection of the software units that commucicate with the controllers or peripheral devices to carry out operations on peripheral devices attached to the machine.
file manager The file manager coordinates the use of the machine's mass storage facilities and maintains record of all the files stored in mass storage including the location of each file, which users have access to various files, and which portions of mass storage are available for new files or extensions of existing files.
memory manager A component of an operating system kernel that is charged with the task of coordinating the machine's use of main memory.
virtual memory Virtual memory is the imaginary memory space created by the memory manager shuffling data and programs back and forth between the main memory and the mass storage through the process of "paging".
boot strapping A procedure that transfers the operating system from mass storage into the main memory.
ROM ROM stands for "read-only memory" that consists of nonvolatile memory cells with content that can be read but not altered.
deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
spooling A technique used for granting several processes access to a common device or resource such as a printer.
super user (or administrator) A super user or administrator can create accounts and gain access to the operating systems by identifying him or herself as the administrator or super user by name and password.
multi-factor authentication Multi-factor authentication is using multiple types of evidence to gain access to computer accounts, usually by something the user 'knows' or 'has' or 'is'.

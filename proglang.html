Chapter 6: Programming Languages
Terms Definitions
machine language Programs located in modern computers that consist of sequences of instructions that are encoded as numeric digits.
debugging A tedious process where the errors of the writing in a program are located and corrected before the job can be finished.
assembler Programs that were developed to convert mnemonic expressions into machine language instructions.
assembly language A mnemonic system that collectively represents programs.
translator A program that was written to translate programs expressed in high-level primitives into machine-language programs.
compiler translation programs that were similar to the second-generation assemblers, except that they often had to compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
interpreter A means of implementing third-generation languages. These programs were similar to translators except that they executed the instructions as they were translated instead of recording the translated version for future use.
cross-platform software The term cross-platform refers to programs that are operating-system independent as well as machine independent. These programs can transfer and execute across networks and internets involving different machine designs and different operating systems.
natural language Natural languages can adequately express algorithms with sufficient patience. It has the ability of a computer program to understand human language as it's spoken and written.
formal language A set of finite or infinite strings drawn from a finite alphabet.
machine independence A term applied to software that is not dependent on the properties of a particular machine and can therefore be used on any machine.
high-level programming language Programming languages that are designed to allow humans to write computer programs and interact with a computer system without having to have specific knowledge of the processor or hardware that the program will run on.
object-oriented programming (OOP) A programming process, which is associated with object-oriented paradigm, a programming paradigm that is the most prominent one in today's software development. It is a software system that is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects. Together, these objects interact to solve the problem.
scripting language Scripting languages are the collection of languages that are a subset of imperative programming languages. These languages are typically used to perform administrative tasks rather than to develop complex programs.
data types:
  • integer - The type integer refers to numeric data consisting of whole numbers, probably stored using two's compliment notation. Operations performed on integer data include the traditional arithmetic operations and comparisons of relative size, such as determining whether one value is greater than another.
  • float - The type float (sometimes referred to as 'real') refers to numeric data that might contain values other than whole numbers, probable stored in floating-point notation. Operations performed on data of type float are similar to those performed on data type integer.
  • string - A data type that is used in programming languages and is used to represent text rather than numbers. It is a type value that can be stored in a variable and is a sequence of characters that can containing letters, numbers, symbols, and even spaces.
primitive data type Data types that are included as primitives in a programming language, such as int for integer and char for character. The types integer, float, character, and Boolean are common primitives. Other data types that have not yet become widespread primitives include images, audio, video, and hypertext.
data structure Data structure is the conceptual shape or arrangement of data, and variables in a program are often associated with it.
Python collection data structures
  • list - Lists are used to store multiple items in a single variable. Lists are created using square brackets. Lists are one of four built-in data types in Python used to store collections of data. List items are ordered, changeable, and allow duplicate values.
  • tuple - Tuples are used to store multiple items in a single variable. A tuple is a collection which is ordered, unchangeable, and allows ordered values. Tuples are written with round brackets.
  • set - Sets are used to store multiple items in a single variable. A set is a collection which is unordered, unchangeable, unindexed, and does not allow duplicate values.
  • dictionary - Dictionaries are used to store data values in key: value pairs. A dictionary is a collection which is ordered, changeable, and does not allow duplicates. Dictionaries are written with curly brackets and have keys and values.
function A set of instructions for performing a task that can be used as an abstract tool by other program units.
local variable A variable declared within a function. It can be referenced only within that function
global variable Variables whose scopes are not restricted to a particular part of a program.
source program A program in its original form.
object program The translated version of a program.
